Stable Marriage Simulator, Variant I

For CS70 Fall 2014
By cs70-asj

Instructions

  1. Change settings.
  2. Press generate.
  3. Change trait settings.
  4. Click save.
  5. Control simulation with 3 buttons.
  6. GOTO 1.

Each trait is renamable and customizable. You can use (very basic!) javascript to write your own comparison functions that will be used. The machine will then use this comparison to decide who to keep as the stable marriage algorithm is run. The comparison should return a positive number if a is "better", a negative one if b is "better", and 0 if they are equal. There are options to either run per time segment, step by day, step by step (each proposer goes at a time), and manual.

Example: You give the proposer a trait called "attractiveness". You give the proposee a trait called "attractiveness". You allocate random numbers to each one. And you program it so the function returns the highest attractiveness. For example, function(a,b) { return a.attractiveness - b.attractiveness; }. That is simple, in theory. I am not a UI/UX person.

Also it is not very error-proof right now, so please do not name your custom traits "id", "func", or "priorityList". Actually, there is no error checking at all so please don't have typos in your custom sort functions.
